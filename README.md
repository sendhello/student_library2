Scenario 1: Library Membership & Resource Management System
Assessment 1 (Modules 1–4) Scenario Description
“A Single Member Borrowing Assistant”
Imagine you are working with one library member who wants a personal console based
assistant to track the items they borrow. Since the current library software is outdated and
slow, the member needs a simple tool they can use to:-based assistant to track the items they
borrow. Since the current library software is outdated and slow, the member needs a simple
tool they can use to:
• Enter their personal details (name, birthdate, email, mobile)
• Record every item they borrow
• Record when items are due
• Group items by due date
• Mark items as returned
The system must be built using fundamental Python only:
• variables, input/output, lists, dictionaries
• if/elif/else
• while loops only (no for loop allowed)
• no functions, no classes, no files
This prototype will serve as the foundation for a full system developed in later assessments.
